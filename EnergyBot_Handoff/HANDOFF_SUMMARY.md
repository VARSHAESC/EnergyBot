# 🤝 Project Handoff: EnergyBot Offline Platform

This document serves as the primary entry point for anyone taking over or collaborating on the **EnergyBot** project.

## 📋 Project Identity
- **Name**: EnergyBot (Stadtwerke X Infrastructure Intelligence)
- **Status**: Production-Ready / Offline-Capable
- **Purpose**: AI-driven analysis of gas and water infrastructure with street-accurate geospatial mapping.

---

## 🏗️ Architecture at a Glance
The system is designed for **privacy and performance** using a local-first stack:
- **Frontend**: Streamlit (Python) for rapid dashboarding.
- **RAG Engine**: ChromaDB + Sentence Transformers (Multilingual).
- **Geospatial**: Custom MST algorithms + OSRM (Open Source Routing Machine).
- **Deployment**: Fully containerized with Docker Compose.

---

## 🚀 How to Get Started (Immediate Steps)

For a detailed technical setup, please refer to:
1.  **[README_DOCKER.md](./README_DOCKER.md)**: The fastest way to run the app using Docker.
2.  **[DEVELOPER_HANDOFF.md](./DEVELOPER_HANDOFF.md)**: Deep dive into the code architecture and modules.

### Quick Run:
1. Ensure Docker is running.
2. Run `./setup_osrm.bat` to prepare maps.
3. Run `docker compose up --build -d`.
4. Visit `localhost:8600`.

---

## 📂 Key Files & Directories
- `app.py`: The UI and main coordination logic.
- `rag_engine.py`: The AI "brain" (indexing, searching, and LLM calls).
- `geo_utils.py`: The GIS engine (Excel processing and routing).
- `excel_data/`: The "Source of Truth" for all infrastructure records.
- `osrm_data/`: Processed map data (generated by setup script).

---

## 🔐 Credentials & Security
- **.env**: Not included in the zip for security. Use `.env.template` to create one.
- **API Keys**: Requires a Groq or OpenAI-compatible API key for the LLM features.
- **Admin Login**: Configured via `APP_USERNAME` and `APP_PASSWORD` in `.env`.

---

## 🛠️ Maintenance Tasks
- **Updating Data**: Replace the Excel file in `excel_data/` and click the refresh button in the app sidebar.
- **Updating Maps**: To change the region (currently Duesseldorf), update the URL in `setup_osrm.bat` and re-run it.

---

## 📞 Support & Handoff Context
This project was developed with a focus on **offline reliability** and **GDPR compliance**, ensuring that sensitive utility data never leaves the local environment except for encrypted LLM inferences (if using cloud LLMs).
